# Product Guidelines

## Brand & Voice
- **Tone:** Developer-friendly, concise, technical but approachable
- **Naming:** "OpenCode Configger" — the utility name. Refer to the configured application as "OpenCode"
- **Error messages:** Be specific about what's wrong and how to fix it

## Visual Identity
- Clean, minimal UI — form-based layout with clear section grouping
- Use system theme / dark theme by default to match developer preferences
- Consistent spacing and alignment
- Tooltips for every config key explaining what it does

## UX Principles
- **Discoverability:** All config options visible through navigation, not hidden
- **Safety:** Validate before save, warn about destructive changes
- **Speed:** App should launch instantly, no splash screens or loading
- **Defaults:** Show OpenCode's defaults as placeholders so users know what they're overriding
